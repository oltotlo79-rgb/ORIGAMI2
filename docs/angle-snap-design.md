# 角度スナップ設計

## 目的と範囲

要件`EDT-006`の角度スナップを実装する。これは一度の頂点追加・移動座標を決める作図補助であり、`EDT-008`の永続的な角度制約とは分離する。

初期版は、明示したアンカーを通り、選択した基準方向に対して指定角度をなす2本の無限直線を候補にする。全方位や全辺を検索せず、候補を最大2本に固定してポインター問い合わせを`O(1)`に保つ。

## 角度設定

- 初期値は`45°`とする。
- プリセットは`11.25° / 15° / 22.5° / 30° / 45° / 60° / 67.5° / 90°`とする。
- 任意角は有限な小数値を入力できる。
- 有効範囲は無向直線同士の小さい方の角度として`0 < angle <= 90°`とする。
- `0°`は既存の平行スナップへ任せる。
- 初期版では一度に1角度だけを有効にする。複数角度セットと角度刻みは後続拡張とする。
- 数式入力との統合は`EDT-004/005`で行い、この段階では評価済み数値を使う。

## アンカー

- 頂点追加では、選択中の一意で有限な頂点をアンカーにする。
- 頂点ドラッグでは、ドラッグ開始座標をアンカーにする。
- アンカーなし、空ID、非有限座標では候補を作らない。
- raw座標がアンカーと完全一致する場合は候補を作らない。
- ドラッグ開始時に、アンカー、角度、参照モード、参照辺形状を固定する。

## 参照基準

次の2モードを明示的に選択する。

1. `水平基準`: グローバルX軸`(1, 0)`を使う。初期値とする。
2. `方向参照辺`: 平行スナップと同じ参照辺を使う。

方向参照辺は平行スナップと角度スナップで共有する。

- 平行スナップがOFFでも角度基準には利用できる。
- 方向参照辺モードで参照辺がない場合は候補を作らない。
- 水平基準への暗黙フォールバックは行わない。
- 参照辺の削除・不正化、新規作成、読込では既存仕様どおり参照を解除する。
- ドラッグ中に参照辺の端点を動かしても、開始時の方向を使う。
- 参照辺の端点順を反転しても候補、key、順位を変えない。

## 候補計算

アンカーを`A`、raw座標を`P`、基準の単位方向を`u`、指定角度を`theta`とする。基準方向を時計回り・反時計回りに回転した2方向へ射影する。

```text
u+ = rotate(u, +theta)
u- = rotate(u, -theta)
Q  = A + dot(P - A, u±) * u±
```

- `distance(P, Q) * scale <= 8 px`を候補にする。
- 時計回り・反時計回りのうち、補正距離が短い方を優先する。
- 完全同距離では反時計回りを優先する。
- 最近傍候補が`accept`で拒否された場合は、もう一方を評価する。
- 両方が拒否された場合だけ辺・グリッドへ進む。
- `90°`では2候補が同じ無向直線になるため、1候補へ決定的に縮約する。
- 候補配列、辺配列、頂点配列の順序には依存させない。

## 候補metadata

角度候補は最低限、次を保持する。

- `kind: 'angle'`
- `anchorId`, `anchorPoint`
- 候補生成に使った有限な`rawPoint`（確定時に同じ射影を再計算する）
- canonical化した`angleDegrees`
- `angleSide: 'counterclockwise' | 'clockwise'`
- `referenceKind: 'global-horizontal' | 'edge'`
- 参照辺の場合は`referenceEdgeId`とcanonical端点
- 補正点と`distancePx`
- 決定的な`key`

keyは計算座標を含めず、アンカー、参照、角度、側だけから作る。数値の`-0`は`0`へ正規化する。

```text
global: angle:[anchorId,"global-horizontal",degrees,side]
edge:   angle:[anchorId,"edge",referenceEdgeId,degrees,side]
```

## 数値安定性

- 参照辺端点は座標の辞書順でcanonical化する。
- 端点差は平行スナップと同じ最大成分スケーリングを使う。
- 縮小後の`hypot`で単位方向を作る。
- 回転は有限な`sin`と`cos`を使う。
- raw座標とアンカーの差も最大成分で縮小してから内積し、最後にスケールを戻す。
- 端点差、単位化、回転、内積、射影点、補正距離、画面距離の各段階で有限性を確認する。
- overflowや表現不能時は候補を返さない。
- 候補選択や辺一致へ画面距離用epsilonを混ぜない。
- metadata再検証の外積許容は`Number.EPSILON`由来の演算誤差だけに限定し、絶対座標の大きさで許容範囲を広げない。

## 優先順位

1. 同一点のT字修復
2. 頂点
3. proper交点
4. 中点
5. 水平・垂直
6. 平行
7. 角度
8. 辺
9. グリッド

グループをまたいだ距離比較は行わない。平行・水平・垂直と同じ候補点になる場合は、それら既存の上位候補を優先する。

## トポロジー保護

確定前に次を再検証する。

- pointとtarget.pointの完全一致
- 有限で非負な距離
- 有限なアンカーと角度
- 正しいkey、side、参照モード
- `0 < angleDegrees <= 90`
- pointがアンカーと異なる
- 水平基準には参照辺metadataがない
- 辺基準では対象IDがちょうど1本だけ存在する
- 参照辺が有限かつ非零長である
- canonical端点が完全一致する
- pointが再計算した角度直線上にある
- rawPointから候補生成と同じ数式で再射影した点がpointと完全一致する

検証後は既存の`splitOrAddPoint()`を使う。

- 一意な辺の厳密な内部点なら原子的辺分割へ昇格する。
- 輪郭辺なら紙境界も同時に分割する。
- 端点一致、複数辺一致、重複辺ID、壊れたmetadataは拒否する。
- 一致辺がなければ通常追加する。
- ドラッグは既存仕様どおり座標移動だけとし、自動接続・分割は行わない。

## UI導線

- スナップチップへ「角度」を追加し、初期値ONとする。
- プリセット選択、任意角入力、基準`水平 / 方向参照辺`を用意する。
- 現在値を例`角度 45°・水平基準`のように表示する。
- 方向参照辺がない場合は、線を選択して方向参照に設定する案内を表示する。
- 不正入力中は最後の正常値を維持し、`aria-invalid`とエラーテキストを設定する。
- 参照辺の表示名は「方向参照（平行・角度）」へ広げる。
- 色だけで選択状態やエラー状態を表現しない。

## ガイド

- アンカーから補正点までの角度線
- raw座標から補正点までの短い補正線
- アンカー位置を通る短い基準線
- 基準線から角度線までの小さな円弧
- `角度 +45°`または`角度 -30°`ラベル
- 参照辺基準では既存の破線オーバーレイ
- 射影係数が負の場合は基準rayと角度rayを同時に180°反転し、円弧を実際のanchor→target線へ接続する

Canvasの現行変換は紙座標と画面座標でYの符号を反転しない。基準線と円弧は実際の`mapPaperPoint`と同じ向きへ変換し、将来Y軸規約を変更する場合も画面上の角度線と一致することを回帰確認する。

## 性能

角度候補生成は常に最大2候補とする。

- 全頂点検索なし
- 全辺検索なし
- プリセット全列挙なし
- 参照辺はプロジェクトrevision変更時だけ解決する
- ポインター移動中は解決済み参照を渡す
- クリック確定時のトポロジー検査だけ既存どおり`O(E)`

10,000個の無関係な頂点・辺を渡しても、角度単体の候補生成時間が要素数に依存しないことを試験する。

## 検証項目

- 既定値、独立切替、45°既定角、全プリセット、任意角
- 角度0、90超、NaN、Infinityの拒否
- アンカーなし、空ID、非有限座標、raw座標とアンカーの一致
- 水平基準と斜め参照辺基準の時計回り・反時計回り候補
- 8 px境界、scale、負座標、極小scale
- 完全同距離時の反時計回り優先
- `accept`で片側拒否後の反対側採用
- 両側拒否後の辺・グリッドへのフォールバック
- 参照辺反転時の点、key、metadata一致
- 90°候補の重複排除
- 頂点、proper交点、中点、水平・垂直、平行、辺との優先順位
- 参照辺ID欠落・重複、長さ0、非有限値
- 最大級座標、端点差overflow、射影overflow
- metadataの角度、side、key、端点改ざん拒否
- 通常辺・逆向き辺・輪郭辺との一致時の分割
- 端点、複数辺、重複ID一致の拒否
- 凹形用紙で片側が紙外の場合の反対側採用
- 頂点追加時の選択アンカー
- ドラッグ開始座標、角度、参照形状の固定
- 参照辺端点自身のドラッグ
- 新規・読込・削除後に参照辺モードが暗黙に水平へ変わらないこと
- 10,000要素での`O(1)`回帰

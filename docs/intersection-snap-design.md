# 交点スナップ設計

## 目的

EDT-006の交点スナップを、10,000本規模の展開図でもUIを停止させず、展開図の接続関係を壊さない形で実装する。

交点へ見た目だけ頂点を追加してはならない。通常辺2本が内部で交差する場合は、1個の共通頂点で両辺を同じ編集命令内に分割し、1回のUndo/Redoで完全に復元できることを必須とする。

## 交点の分類

| 分類 | 初期対応 | 処理 |
|---|---|---|
| proper X | 対応 | 新しい共通頂点1個で通常辺2本を原子的に分割する |
| T字 | 後続 | 既存の端点頂点を再利用し、相手側の内部辺だけを分割する |
| 同じ頂点IDを共有する端点 | 候補外 | 既に接続済みのため通常の頂点スナップを使う |
| 同じ座標にある別IDの端点 | 候補外 | 暗黙に統合せず、将来の頂点統合命令で修正する |
| 正の長さを持つ共線重複 | 候補外 | 単一の交点ではないため、重複解消操作へ分離する |
| 非有限・長さ0・計算不能 | 候補外 | 診断対象とし、交点操作は行わない |
| 輪郭辺を含む交点 | 後続 | Paper境界も同時更新する専用のsheet-aware命令を使う |

幾何分類には画面ピクセルの許容誤差を混ぜない。ピクセル距離は、正確に求めた候補をカーソルが選ぶ閾値にのみ使う。

## データと責務

フロントエンドの候補は、交点座標、決定的なキー、canonical順の2辺ID、各辺の始点から終点に沿ったfractionを持つ。単一辺用の`sourceId`へ交点情報を詰め込まない。

ただし、フロントエンドの座標やfractionは編集結果の正本にしない。Rustコアは受け取った2辺IDから交差を再計算し、proper Xであることを再検証する。

バックエンド応答はプロジェクトスナップショットに加えて、操作後に選択すべき交点頂点IDを明示する。将来のT字対応では新規頂点が増えないため、スナップショットのID差分だけに依存しない。

## 原子的な編集命令

proper Xを既存の辺分割命令2回で処理してはならない。異なる頂点IDの生成、2回目だけの失敗、中間状態の露出、Undo単位の分裂が起こるためである。

専用命令は変更前に次をすべて検査する。

- project IDとrevision
- 2つの異なる対象辺IDと、その各IDの一意性
- 非輪郭辺であること
- 各端点頂点レコードの存在と一意性
- 有限な座標と、両辺の厳密な内部にある単一点交差
- 新規頂点IDと2つの新規辺IDの一意性
- 交点座標を占有する既存頂点がないこと

成功時は元辺ID・向き・線種を各前半へ残し、後半辺を決定的な位置へ挿入する。失敗時はpattern、paper、revision、dirty、Undo/Redo履歴を一切変更しない。

## 性能設計

10,000本に対する全組合せ約5,000万pairの走査は禁止する。

線分のAABBをrevision単位の空間索引へ格納し、カーソル周辺の小さな検索矩形に交差する線分だけを取り出してnarrow phaseを行う。索引はドラッグプレビューごとに再構築しない。候補pairは辺ID順にcanonical化して重複計算を避ける。凹多角形では紙内判定をランキング前に適用し、最近傍が紙外なら次の紙内候補を選ぶ。

病的に多数の線が一点へ集中するデータでは出力候補自体が二乗になるため、1回の問い合わせで調べるpair数または処理時間に上限を設ける。上限到達時は途中までの不完全な最近傍を採用せず候補なしを返し、UIを固めず候補過密としてズームを促す設計へ拡張する。

初期目標は、10,000本の疎なデータでカーソル問い合わせp95を2 ms以下、メインスレッドの1フレーム処理を8 ms以下とする。実Canvas計測後に正式値を更新する。

## 候補優先順位

初期のproper X対応では、既存の頂点スナップを交点より優先する。T字対応時は、既存頂点を選ぶだけでは内部辺が未分割のままになるため、同一点のT字修復候補だけを通常頂点より優先する。

目標順序は次のとおり。

1. 同一点のT字修復
2. 頂点
3. proper交点
4. 中点
5. 水平・垂直
6. 平行
7. 角度
8. 辺
9. グリッド

方向スナップは作図のアンカーや参照辺が必要なため、単純な点候補と分けたコンテキストを持たせる。

## 実装段階

1. proper Xだけを分類する純粋関数、局所空間索引、候補型、10,000本性能テストを追加する。UI接続はまだ行わない。
2. Rustコアへ通常辺2本のproper X原子分割命令を追加し、異常系とUndo/Redoを網羅する。
3. 交点トグルとガイドを追加し、頂点追加クリックだけを原子命令へ接続する。頂点ドラッグにはまだ適用しない。
4. T字で既存頂点を再利用する処理と優先順位を追加する。
5. 輪郭辺との交点、3本以上が同一点に集まる場合の一括接続を追加する。
6. 水平・垂直、平行、角度スナップを順に追加する。

各段階で極端値、逆向き辺、決定性、失敗時原子性、1回のUndo、成功頂点の選択、10,000本性能を回帰検証する。

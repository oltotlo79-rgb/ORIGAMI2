## 2. 目的

### 2.1 初期目的

折り紙作家が、一枚の紙の展開図を精密に作成し、厚さを考慮した3D折りシミュレーションによって完成形と折り過程を検証し、折り手順を制作できるデスクトップアプリケーションを提供する。

### 2.2 長期目的

初心者が参考画像、3Dモデル、部品構成または骨格から作りたい形を指定し、展開図と折り手順を端末内で自動生成できる設計支援アプリケーションへ発展させる。

### 2.3 最優先品質

幾何学的な正確さと折り可能性の判定を最優先とする。操作性、表示品質、処理速度は重要であるが、計算結果の正しさを損なう近似を無断で確定結果として扱わない。

## 3. 製品方針

- 初版の正式対応OSはWindows 10/11とする。
- macOSは自動ビルド、テストおよび`.app`生成のCI検証だけを維持する。初版では実機動作、正式配布および利用者サポートを保証しない。
- 基本機能および将来の自動設計を端末内で実行する。
- 作品データ、利用統計、クラッシュ情報を外部へ自動送信しない。
- サーバーの維持費や有料APIを必要としない。
- ソースコードとリリースをGitHubで公開する。
- GPL-3.0を第一候補とし、商用利用、改造、再配布を許可する一方、配布される派生物には同ライセンスに基づくソース公開を求める。
- Windows実機E2Eと性能ベンチマークはオーナーが実施し、合否結果と計測値を受け入れ証跡として提供する。
- 外部試用版は、初版の全必須機能が完成した後に少人数の折り紙作家へ提供する。
- 開発中は機能単位の内部版を作成し、利用者が都度確認する。

## 4. 対象ユーザー

### 4.1 初版

- 主利用者：折り紙作家および高度な展開図を設計する利用者
- 試用者：開発依頼者および少人数の折り紙作家
- UI言語：日本語、英語

### 4.2 将来版

- 折り紙設計の専門知識を持たない初心者
- 画像や3D形状から作品を作りたい利用者

## 5. スコープ

### 5.1 初版の必須範囲

1. 一枚紙の2D展開図編集
2. 精密作図、数式入力、幾何制約
3. 中央面基準の近似方式で紙厚を考慮した3D表示と折り操作
4. 衝突検出および衝突前停止
5. 局所検証と、時間制限つき全体平坦折り判定
6. 直線の折り線をまたぐ複数層の折り重ね操作
7. 折り手順の編集、記録、再生
8. 指先・つまみ位置等の簡略3Dガイド
9. 画像およびPDFの折り図出力
10. 指定された2D・3Dファイル形式の入出力
11. 日本語・英語、テーマ、レイアウト、ショートカット設定
12. 自動保存、クラッシュ復旧、永続Undo/Redo
13. 折り線10,000本以上を想定したデータ構造と編集・表示性能

### 5.2 将来範囲

- 動物・昆虫を対象とした完成形からの自動設計
- 参考画像および3Dモデルからの形状・骨格・部位認識
- 部品構成・骨格編集による目標形状指定
- 展開図と折り手順の完全自動生成
- 膨らみ、突起、関節および可動範囲を含む目標形状制約
- 評価基準と難易度条件を考慮した候補生成
- ペンタブレット、タッチ操作
- GIF、動画、対話型Web形式の手順出力

### 5.3 初版の対象外

- 複数枚の紙
- 接着
- 曲線折りおよび曲線切断
- 折り曲げ半径、紙の弾性、繊維方向、塑性変形の物理計算
- 紙厚に起因する厳密な層ずれの再現
- 層を個別選択してめくる操作（中割り折り等）
- リアルな手・腕の3D表示
- データ収集型テレメトリー
- クラウド計算および運営サーバー

## 6. 機能要件

要件IDの優先度は、`MUST`が初版必須、`FUTURE`が将来要件を表す。

### 6.1 プロジェクトと紙

| ID | 優先度 | 要件 |
|---|---|---|
| PRJ-001 | MUST | 一作品につき、元が一枚である紙を管理できること。 |
| PRJ-002 | MUST | 紙の輪郭を、直線辺からなる任意の単純多角形として定義できること。正方形・長方形に限定しない。 |
| PRJ-003 | MUST | プロジェクトごとに「切断を許可する／許可しない」を設定できること。 |
| PRJ-004 | MUST | 切断許可時は切断線を使用でき、切断禁止時は切断線の作成および切断状態への遷移を禁止すること。 |
| PRJ-005 | MUST | 接着を扱わないこと。 |
| PRJ-006 | MUST | 紙の厚さとして0以上の一定値を設定できること。新規作成時の既定値は0.10 mmとし、数値入力の上下ボタンは1操作につき0.01 mm増減する。直接入力した値は0.01 mmへ強制丸めしない。 |
| PRJ-007 | MUST | 紙の表裏に、個別の単色または画像・模様を設定できること。 |
| PRJ-008 | MUST | 長さの単位をmm、cm、inch、紙の辺を基準とする比率から選択できること。 |
| PRJ-009 | MUST | 頂点、折り線、面に安定した内部IDを付与し、名前、色、メモを設定できること。 |

### 6.4 検証と折り可能性

| ID | 優先度 | 要件 |
|---|---|---|
| VAL-001 | MUST | 頂点・線の重複、線の交差、未分割交点、閉じていない面、輪郭外要素、接続関係を検証すること。 |
| VAL-002 | MUST | 各頂点について川崎定理、前川定理等の局所的な平坦折り条件を検証すること。適用条件外の場合はその旨を示すこと。 |
| VAL-003 | MUST | 展開図全体の平坦折り可能性を設定された時間制限内で判定し、結果を「可」「不可」「不明」の3値で返すこと。時間切れ、処理上限到達または証明不足は「不明」とし、「不可」と推測しないこと。判定に用いた面の層順序を後続の3D操作で利用できる形で管理すること。 |
| VAL-004 | MUST | 3D折り操作の経路上で衝突を検出した場合、最初に検出した衝突の直前で操作を自動停止し、衝突した面、位置および理由を表示すること。初版は任意の無衝突経路を完全探索することを要件としない。 |
| VAL-005 | MUST | 平坦折り判定中は、完了、時間制限到達または利用者による中止まで処理状態と進捗を表示すること。 |
| VAL-006 | MUST | 平坦折り判定を利用者がいつでも安全に中止できること。中止によって編集データを失わないこと。 |
| VAL-007 | MUST | 判定処理をバックグラウンドで実行し、UIを応答可能に保つこと。 |
| VAL-008 | MUST | VAL-004により停止した3D操作を衝突状態まで実行せず、停止理由と可能な修正候補を表示すること。 |
| VAL-009 | MUST | 「不可」「可」「不明」「利用者による中止」「計算エラー」を区別すること。時間切れ、中止または計算エラーを「不可」として扱わないこと。 |

> 注記：全体平坦折り判定は停止性を優先し、時間制限内に証明できない入力を正式に「不明」とする。判定方式、証明情報、対象クラスおよび並列化方式は設計で定義するが、未証明の結果を「可」または「不可」と表示してはならない。「利用者による中止」と「計算エラー」は判定値ではなく実行の終了状態であり、3値のいずれにも変換しない。一般的な折り経路の完全探索は初版の対象外である。

### 6.5 3D折りシミュレーション

| ID | 優先度 | 要件 |
|---|---|---|
| SIM-001 | MUST | 2D展開図から頂点、辺、面およびヒンジ関係を構築して3D表示できること。 |
| SIM-002 | MUST | 折り線を選択し、角度スライダーおよび数値入力で操作できること。 |
| SIM-003 | MUST | 3D上の紙をマウスでつかんで折る操作ができること。 |
| SIM-004 | MUST | 利用者が固定面を選択し、折り線の反対側につながる面群を連動させられること。 |
| SIM-005 | MUST | 紙の厚さを表示と面同士の衝突・重なり判定に反映すること。 |
| SIM-006 | MUST | 衝突直前で操作を自動停止し、衝突した面、位置、原因を表示すること。 |
| SIM-007 | MUST | 紙の表裏の色・模様を3D表示へ反映すること。 |
| SIM-008 | MUST | 切断後も各部分が元の一枚紙に由来することを保持し、切断された接続関係を正しく反映すること。 |
| SIM-009 | MUST | 折り線10,000本以上のデータを読み込み、表示・選択・基本編集できる設計とすること。具体的な応答時間は性能設計で定義する。 |
| SIM-010 | MUST | 折り畳んだ現在の3D状態に一直線の新しい折り線を指定し、その線をまたいで重なっている複数層をまとめて折れること。結果を展開図へ層ごとの山谷割当てを持つ対応折り線として自動追加し、1回の操作を折り手順タイムラインの1ステップとして記録して段階再生できること。実装はVAL-003で導入する層順序管理と、共有関係×接触・交差種別の衝突分類全組合せ表および境界回帰の完成後に行うこと。 |

> 初版の紙厚モデルは`centered_mid_surface_v1`を正式仕様とし、各面の中央面を運動の正本として入力紙厚の半分ずつを表裏へ押し出す。これは衝突・重なりを判定するための近似であり、紙厚に起因する厳密な層ずれは将来課題とする。
>
> SIM-010の初版対象は一直線の折り線をまたぐ層をまとめて折る操作に限る。層を個別選択してめくる操作や、中割り折り等の技法固有の層選択運動は将来課題とする。INS-008の名前付き技法は、これらの物理操作が初版で自動実行できることを意味しない。

## 8. 非機能要件

### 8.1 正確性

- 幾何計算には用途に応じた頑健な数値処理を用いる。
- 数式入力の原文を失わず、高精度評価値と関連付ける。
- 許容誤差を一箇所で管理せず、作図、トポロジー判定、衝突判定、表示に適した値を明示する。
- 判定結果には、可能な限り根拠、対象要素、誤差条件を付ける。

### 8.2 性能

- 一般的な家庭用PCを基準とする。
- 基本的な2D編集は専用GPUを必須としない。
- 3D表示および適用可能な計算にはGPUを利用し、利用できない環境では代替処理を提供する。
- 折り線10,000本以上のプロジェクトで、読み込み、表示、選択、基本編集が可能であることを目標とする。
- 全体平坦折り判定は設定された時間制限内で実行し、UIをブロックせず中止可能にし、時間切れは「不明」とする。
- 具体的な基準PC、データセット、応答時間、メモリ上限は性能設計書で確定する。

### 8.3 可搬性

- 初版の正式動作検証と配布はWindows 10/11を対象とすること。
- プロジェクトファイルをOS非依存とし、macOS自動CIでも読込・保存の互換性を検証すること。
- OS固有処理を計算コアから分離すること。
- 将来のLinux、Web、タブレット対応を妨げないデータモデルとすること。ただし初版では保証しない。

### 8.4 信頼性

- 長時間計算の中止、アプリ異常終了、保存失敗で既存の正常なプロジェクトを破損しないこと。
- 保存時は一時ファイルと置換等を用い、途中書き込みによる破損を防ぐこと。
- 未知フィールドを含むプロジェクトの扱いとバージョン移行方針を定義すること。

### 8.5 プライバシーとセキュリティ

- 作品データを端末外へ自動送信しない。
- 利用統計およびクラッシュレポートを自動送信しない。
- 読み込むSVG、画像、3Dモデル、プロジェクトファイルを信頼できない入力として検証する。
- 更新確認以外のネットワーク通信を初版の通常機能に必要としない。

### 8.6 保守性

- 幾何・トポロジー・制約・シミュレーション・ファイル変換・UIを分離する。
- 計算コアはUIから独立して自動テスト可能にする。
- 日本語・英語の文言をソースコードへ直接埋め込まず、翻訳資源として管理する。
- ファイル形式にはバージョン番号を持たせ、移行処理を提供する。

## 11. 初版の受け入れ条件

初版は、次の条件をすべて満たした時点で外部試用可能とする。

1. 本書の`MUST`要件が実装済みである。
2. Windows 10/11で`.ori2`ファイルを作成・保存・再読込でき、Undo/Redo両方が非空かつ履歴上限が既定値でないprojectも、再読込後に同じ履歴順序と上限を利用できる。macOS CIでも同一形式の自動往復試験を通過する。
3. 任意の直線多角形の紙に展開図を作成し、切断可否の設定が機能する。
4. 数式入力、スナップ、指定された幾何制約、リアルタイム対称編集が機能する。
5. 3D移行前検証が不正データを検出し、問題要素と理由を示す。
6. 折り線の数値操作と3Dドラッグ操作が機能し、固定面を選択できる。
7. 設定した紙厚が中央面基準近似による表示および衝突判定へ反映され、衝突前に理由付きで停止する。
8. 全体平坦折り判定が時間制限内で「可」「不可」「不明」を正しく区別し、未証明または時間切れを「不可」としない。
9. 長時間計算中もUIが応答し、安全に中止できる。
10. 複数折り線、固定面、持ち位置および一直線の折り重ね操作を含む手順を作成・記録・編集・再生でき、折り重ねで生じた層ごとの山谷折り線が展開図へ追加される。
11. 手順画像とPDFを出力できる。
12. 指定された2Dおよび3D形式の入出力試験を通過する。
13. 自動保存後のクラッシュを模擬し、未保存状態を復元できる。
14. 既知の基本展開図による自動テストと、実物の紙による比較検証を通過する。
15. オーナーがWindows実機で標準性能試験用の10,000本以上のデータを計測し、提供した結果が性能設計書の基準を満たす。
16. アプリが作品データ、利用統計、クラッシュ情報を外部送信しないことを確認する。
17. Windows 10/11で日本語・英語、ライト・ダーク、主要ショートカットが機能する。
18. macOSの自動ビルド、テストおよび`.app`生成がCIで成功する。実機での操作確認は検証環境を用意した後の将来受け入れとする。

## 12. 検証方針

### 12.1 自動検証

- 幾何プリミティブの単体テスト
- 頑健な交差・包含・向き判定テスト
- 制約充足・矛盾検出テスト
- 川崎・前川条件の既知例と反例
- 面抽出、切断、トポロジー更新の性質テスト
- 順運動学・逆操作の再現性テスト
- 厚さを考慮した衝突判定テスト
- 既知の平坦折り可・不可パターン
- 時間制限到達時に「不明」を返し、「不可」と混同しない平坦折り判定テスト
- 折り重ね操作の層順序、層ごとの山谷割当て、展開図更新およびタイムライン再生テスト
- ファイル形式の往復変換テスト
- Undo/Redo両stack・非既定履歴上限の保存再読込、legacy 2-entry互換、全command/inverseの意味再生テスト
- hash破損、別projectへの履歴付替え、意味不一致、およびUndo/Redo後にcurrentとなる不正instruction poseを拒否し、既存projectを変更しないテスト
- 保存中断・クラッシュ復旧テスト
- 10,000本以上の性能回帰テスト

### 12.2 実物検証

- 基本折り、伝承作品、既知の複雑作品を実際の紙で折る。
- アプリの面位置、重なり順、停止位置、完成形と実物を比較する。
- 差異を計算不具合、紙厚モデル、弾性・塑性等の対象外物理要因に分類する。

### 12.3 試用

- 全必須機能完成後、GitHub Releasesで限定的なテスト版を配布する。
- 初版のWindows実機E2Eと性能ベンチマークはオーナーが実施し、結果と計測値を受け入れ記録へ添付する。
- macOS実機試験は検証環境を用意できた時点で別途実施し、初版Windows正式版の公開を遮断しない。
- 少人数の折り紙作家がGitHub Issuesへ問題・要望を登録する。
- 作品ファイルやログは試用者が内容を確認し、明示的に添付した場合だけ共有する。

## 13. 開発上の主要リスク

| リスク | 影響 | 対応方針 |
|---|---|---|
| 任意展開図の全体平坦折り判定 | 完全判定が現実的時間で終了しない可能性 | 時間制限つき3値判定とし、証明不足・時間切れを「不明」として可否から分離する |
| 3D折り操作中の衝突 | 離散サンプル間の衝突を見落とす可能性 | 連続衝突判定で最初の衝突直前へ停止し、対象面・位置・理由を表示する |
| 紙厚とヒンジの矛盾 | 厚さゼロの折り軸では物理的に重なる | 初版は中央面基準近似を正式仕様とし、厳密な層ずれ、弾性、曲げ半径とは区別する |
| 10,000本以上の編集と3D | メモリ・描画・判定性能低下 | 空間索引、差分更新、GPU描画、段階的詳細度、性能回帰試験 |
| 多形式の互換性 | 情報欠落や意味の差異 | 変換能力表、警告、往復試験、独自形式を正本とする |
| 全機能完成後まで外部試用しない | 作家の操作要求を発見する時期が遅れる | 依頼者による機能単位の内部確認を必須とする |
| macOS実機を検証できない | OS固有の操作・表示不具合を初版で検出できない | 自動ビルド・CI検証だけを維持し、実機環境を用意した後に正式対応を別途判定する |
| GPL依存関係 | 非互換ライセンス混入 | 依存追加時にライセンス検査を行う |

## 14. オーナー決定と残る設計事項

2026-07-18に、次を初版の正式仕様として決定した。

1. 紙厚は中央面基準の近似方式`centered_mid_surface_v1`を使用し、厳密な層ずれは将来課題とする。
2. VAL-003は時間制限つきの「可／不可／不明」判定、VAL-004は衝突直前の自動停止と理由表示とし、一般的な折り経路の完全探索を初版から外す。
3. 初版の正式対応OSはWindows 10/11とし、macOSは自動ビルド・CI検証だけを維持する。
4. Windows実機E2Eと性能ベンチマークはオーナーが実施し、結果と計測値を受け入れ証跡として提供する。
5. SIM-010「折り重ね操作」を初版MUSTへ追加し、VAL-003の層順序管理完成後に実装する。

2026-07-19に、SIM-010は衝突分類の上に載る機能であるため、共有関係×接触・交差種別の全組合せ表、共有頂点・共有ヒンジの許容境界、貫通・接触・判定保留の回帰を先に固定し、その完了前に折り重ねUIを実装しないことを追加決定した。

以下は、確定済みの要求を変えない範囲で設計または技術検証により決定する。

1. UIフレームワーク、実装言語、幾何計算ライブラリ
2. `.ori2`の内部圧縮・スキーマ・バージョン移行方式
3. 平坦折り判定で証明可能な対象クラス、証明情報および時間・資源上限
4. 10,000本性能試験の基準PCと具体的な応答時間
5. DXF、OBJ、STL、glTF、FOLD各形式の対応バージョンと変換能力
6. PDF折り図のページ構成と図記号規約
7. 正式製品名および商標確認
8. GPL-3.0-onlyまたはGPL-3.0-or-laterの最終選択

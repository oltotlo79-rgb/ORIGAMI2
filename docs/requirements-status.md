# MUST要件 実装状況

更新日: 2026-07-18

`requirements-definition.md`のMUST 86件を、利用者がUIから実行できるかを基準に追跡する。内部基盤やテストだけが完成していても、UI未接続なら「実装済み」へは上げない。

- 実装済み: 要件の主要な利用経路がUIから動作する
- 部分実装: 有用な一部が動作するが、要件に明記された範囲を満たさない
- 未着手: 要件固有の利用経路が存在しない

現在の行単位集計は **実装済み20 / 部分実装23 / 未着手43**。第三者監査本文の集計値「21 / 23 / 42」は、同レポート内の各行判定を合計した値と1件ずれるため、以下の行単位判定を正本とする。

## 更新ルール

各機能checkpointで該当IDだけを更新し、状態変更の根拠となるUI経路、保存形式、test、制限を短く追記する。内部品質だけの変更では状態を上げない。要件自体を変更する場合は`requirements-definition.md`と本表を同じcommitで更新する。
